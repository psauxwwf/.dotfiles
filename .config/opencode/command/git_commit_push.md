Add commit and push this changes

1. git commit
2. git push

Use this commit prefix:

- feat: – new functionality, feature (something that adds value to the user).
- fix: – fix a bug or error.
- docs: – changes only in the documentation.
- style: – edits style, formatting, spaces, commas, etc., without changing the logic.
- refactor: – code refactoring without adding functionality and without fixing bugs.
- perf: – performance improvements.
- test: – adding/changing/correcting tests.
- build: – changes in the project build, dependencies, build tools.
- ci: – changes in CI/CD configuration (GitHub Actions, GitLab CI, etc.).
- chore: – other service changes (cleaning, updating dependencies, minor tasks that do not affect functionality).
- revert: – rollback a specific previous commit.
